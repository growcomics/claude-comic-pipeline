# Optional Flow-driving checklist for real-model SFX calibration

The PIL matrix in `contact-sheet.png` is a deterministic visual calibration —
useful for picking a level, but the action lines / SFX are drawn algorithmically
rather than generated by the rendering model. If you want real Nano Banana 2
output for fidelity verification, drive Flow manually with this checklist.

**Why Claude can't drive Flow from here:** Flow's prompt input is a Slate.js
editor that only accepts trusted browser InputEvents. The MCP tools available
in this session (Chrome MCP `javascript_tool`, computer-use on Chrome) can
neither dispatch trusted events nor synthesize Slate's internal state
updates in a way that the React component's submit handler accepts.

## Setup (do once)

1. Open the source bicep in Flow:
   https://labs.google/fx/tools/flow/project/f66c66c5-21ca-46c4-9e89-133e9100f6ba/edit/01d2f684-8494-45b0-a701-c50862c16126
2. Confirm the model selector reads `🍌 Nano Banana 2 x4`.

## Per-variant procedure

For each variant below: paste the prompt into the "What do you want to change?"
field, click the `→ Create` button, wait for the 4 outputs to render, pick the
one closest to the level description, click `download`, and rename the file to
`variants/<variant_id>-flow.png` in
`skills/comic-production/references/sfx-calibration/`.

### A-axis — action lines, no SFX

- **A0-flow** *(skip — equals source bicep)*
- **A1-flow** — subtle action lines
  ```
  Composite a SUBTLE 2D comic-book action-line overlay on top of this photoreal
  bicep flex: 4-6 thin radial lines emanating outward from the bicep peak, low
  contrast, light grey. The underlying photoreal bicep, skin tone, costume, and
  speech bubble must remain UNCHANGED. Lines sit ON TOP as a clear comic-style
  layer. No SFX text.
  ```
- **A2-flow** — medium action lines
  ```
  Composite a MEDIUM-DENSITY 2D comic-book action-line overlay: 10-12 radial
  lines emanating from the bicep peak, mix of thin and bold strokes, clearly
  readable as an energy burst but not dominant. Photoreal bicep, costume, and
  speech bubble unchanged. No SFX text.
  ```
- **A3-flow** — heavy action lines
  ```
  Composite a HEAVY 2D comic-book action-line overlay: 18-25 dense radial-burst
  lines, bold strokes with slight chromatic-aberration accent (faint cyan and
  magenta fringing). Bicep stays photoreal underneath. No SFX text.
  ```
- **A4-flow** — overdone
  ```
  Composite an OVERDONE 2D comic-book action-line overlay: 30+ chaotic lines,
  multiple impact-burst centers overlapping, almost obscuring the bicep.
  Bicep still visible underneath but action lines dominate. No SFX text.
  ```

### B-axis — SFX text, no action lines

- **B0-flow** *(skip — equals source bicep)*
- **B1-flow** — subtle SFX
  ```
  Composite a single small SFX word "*flex*" in lowercase italic, light grey,
  low alpha, placed in the bottom-right corner of this photoreal bicep panel.
  Subtle — integrated tonally. No action lines.
  ```
- **B2-flow** — medium SFX
  ```
  Composite a single bold SFX word "FLEX" in classic comic-book burst styling
  (yellow fill, thick black outline, slight tilt, drop shadow), sized at
  approximately 15% of the frame height, placed behind/beside the bicep — not
  on top of it. Photoreal bicep unchanged. No action lines.
  ```
- **B3-flow** — heavy SFX
  ```
  Composite a bold SFX word "FLEX" dominating the frame, ~30% of frame height,
  yellow fill with dark-red outline and heavy drop shadow, slight tilt, classic
  comic-book burst styling. Bicep still visible underneath. No action lines.
  ```
- **B4-flow** — overdone stacked
  ```
  Composite multiple stacked SFX words: "FLEX!", "POMF!", "GRRRR!" — varying
  sizes, dominating the panel, photoreal almost lost. Classic comic-burst
  styling per word. No action lines.
  ```

### C-axis — combined

- **C1-flow** — A1 + B1: subtle both
- **C2-flow** — A2 + B2: medium both (likely sweet spot)
- **C3-flow** — A3 + B2: heavy lines, medium SFX (peak-flex variant)

## After

Drop the picked-variant files into `variants/` alongside the `*-flow.png`
naming, and re-run `python3 generate_matrix.py` (the script will skip files
that already exist if you add that guard — or, simpler, just look at the
side-by-side manually).
