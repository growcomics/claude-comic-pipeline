# Validation A/B — refs-are-truth refactor

## Setup

Generated by calling `compose_prompt()` against an intact shotlist
(`projects/chun-li-test/shotlist.json`) at peak conditions (lineup +
tier reinforcement + env ref + prior anchor attached).

Why chun-li-test and not the Yuna project the user named:
- The Yuna project ships without a `shotlist.json` (only
  `final/comic.pdf` + 5 accepted panel PNGs survived).
- That's PRECISELY the persistence-gap bug this refactor closes. Going
  forward every accepted panel writes `prompt.txt`,
  `attached_refs.json`, and `panel-plan.json` next to the image so the
  exact prompt and refs are reconstructable.
- For the A/B comparison, chun-li-test gives us a real shotlist with a
  representative tier-6 stage-change panel — the worst-case scenario
  for the old composer's appearance bloat.
