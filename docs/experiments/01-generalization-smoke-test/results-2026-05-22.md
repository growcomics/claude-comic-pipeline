## Per-failure analysis

### SOFT-FAIL #1 + #2 ‚Äî container-shape disagreement (`chun-li-serum-courtyard`, `Mira's Story ‚Äî Ch1 Rooftop Pool`)

Both projects use **Shape B** ‚Äî a flat `panels` array at the shotlist root ‚Äî while the other 13 projects use **Shape A** ‚Äî root-level `pages: [{panels: [...]}]`. `next_panel.py` walks `pages[*].panels[*]`. When the root has no `pages` key, the walk yields an empty iteration, and the script returns:

```json
{"next_panel": null, "message": "All shotlist panels have an accepted version. Nothing pending.", "accepted_count": 0}
```

That message is a lie. Verified by:
- `chun-li-serum-courtyard/shotlist.json` has 12 panels under root `panels`; none have a `state`, `status`, `accepted`, `version`, or `accepted_version` field; no `pages/` or `final/` directory exists on disk.
- `Mira's Story ‚Äî Ch1 Rooftop Pool/shotlist.json` has 24 panels under root `panels`; same acceptance-marker absence.

**Magnamus framing:** the recent fixes addressed *field*-level vocabulary disagreement (dialogue speaker keyed as `character` vs `speaker` ‚Äî `961f9b5`). The remaining gap is one layer up: **container**-level shape disagreement. The walker assumes a `pages` envelope. Two of the user's real projects use a flat layout ‚Äî likely produced by an older or alternate path through `script-breakdown`, since they pre-date the standardization on `pages`.

The failure mode is the most dangerous variety of schema disagreement: **silent false-positive**, not crash. A user running `next_panel.py` on either project sees "all done" and stops, rather than seeing a stack trace and investigating.

Not a fault, but worth confirming the doubled counts in a future pass since most other projects have `acc < page_count` or `rem ‚â™ acc`.

## Top-3 next fixes (prioritized by projects unblocked)

1. **Container-shape adapter for flat-`panels` shotlists** *(unblocks 2 projects)*
   Teach `next_panel.py`'s panel-discovery path to also walk root-level `panels[]` when `pages` is absent. Both flat-`panels` shotlists are real, in-progress comic work (`Mira's Story ‚Äî Ch1` is a current user project; `chun-li-serum-courtyard` is the courtyard chapter). Without this fix, neither can advance through the comic-production pipeline.

2. **Loud failure on zero-panel walks** *(defensive, unblocks future variants)*
   When the panel walker finds 0 candidates under its expected container key, emit a warning to stderr (or a non-pending exit code) rather than the cheerful "All shotlist panels have an accepted version" message. The current quiet path is what made fix #1 invisible until this experiment. Even with #1 landed, the next container variant should fail loudly.

3. **Extend `script-breakdown/scripts/validate_shotlist.py` to cover container shape** *(prevents new schema drift)*
   The validator (added in `961f9b5`) currently checks dialogue-shape conformance. Extend it to assert root shape: either `pages: [{panels: [...]}]` OR `panels: [...]` ‚Äî and warn on mixed/ambiguous shapes. Surfaces the shape variant at authoring time so it never reaches `next_panel.py` silently.

Fix #1 alone closes the hypothesis for the two affected projects; #2 and #3 are defenses against the next variant.

## Verdict on the hypothesis

**Partially confirmed.** The schema-disagreement fixes on `main` generalize at the *crash-or-no* layer ‚Äî every one of 15 real projects produces well-formed JSON without exception. They do **not** generalize at the *semantic-correctness* layer ‚Äî 2/15 projects (13 %) are silently miscategorized as complete because the walker is bound to a single container shape.

Composition layer is stable for shotlists that match the `pages ‚Üí panels` shape; one layer up, container-shape disagreement is the next frontier.
